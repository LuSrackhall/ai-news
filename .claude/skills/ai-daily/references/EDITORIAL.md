# AI 日报 — 内容标准

## 文章结构

```
# AI 日报 | {date}
> {hook — 一句话钩子，制造冲突或数据冲击}

## 今日速览
- **{title}**：{one_liner — 30字以内}

## 重磅深度（1-3 条）
### {title}
**发生了什么**：{事实，不含观点}
**技术/商业细节**：{数据、参数、金额}
**为什么重要**：{影响范围、行业意义}
**意味着什么**：{趋势判断、预测}
> 来源：[source](url)

## 重要动态（3-6 条）
### {title}
**要点**：{核心信息}
{分析 — 2-3 句}
> 来源

## 快讯（3-8 条）
- **{title}**：{一句话事实}

## 编辑观点
{观察 — 今天发生了什么}
{证据 — 用数据支撑}
{判断 — 这意味着什么}
{预测 — 接下来会怎样}

---
*数据来源: {sources} | AI 辅助生成，经审核*
```

## 播客脚本结构（双人对话）

```
[开场 Hook · 10-20s] — 男女主播对话开场，制造悬念
[今日概览 · 15-30s] — 概括今天最重要的 3 个点
[重磅新闻 · 每条 30-60s] — 双人对话深度解读
[快速浏览 · 每条 10-20s] — 快节奏对话过信息
[收尾 · 10-20s] — 总结 + 趋势判断
```

每段为对话数组：`[{ speaker: "M"|"F", text: "...", duration_s: N }]`

**总时长：180-300 秒**
**格式：** TTS 友好（无括号注释、无表情、无舞台指示）

## 写作风格

### 必须做到

- **数据锚定** — 每条深度新闻必须有具体数字（金额、参数、百分比）
- **来源标注** — 每条新闻必须标明来源
- **口语化** — 播客脚本用短句，避免书面语
- **冲突制造** — hook 必须有对比或反差
- **去形容词** — 少用"重大""重要""值得关注"，用事实说话

### 禁止（反模式）

- **空洞表述** — "值得关注""意义深远""引发热议"（无信息量）
- **编造数据** — 文章中出现原始数据里没有的数字
- **编造 URL** — 文章中出现 curated items 里没有的链接
- **重复信息** — 速览和深度部分重复同一条新闻的相同信息
- **PPT 风格** — 纯文字堆砌，没有数据支撑

## editorial 四段结构

1. **观察** — 今天发生了什么（事实层面）
2. **证据** — 用数据支撑观察（数字、对比）
3. **判断** — 这意味着什么（分析层面）
4. **预测** — 接下来会怎样（前瞻层面）

每段至少 30 字。四段之间有逻辑递进关系。
