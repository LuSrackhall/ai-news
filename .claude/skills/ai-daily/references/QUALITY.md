# AI 日报 — 质量标准

## 自动检查项（校验 Policy）

### 1. 幻觉 URL 检测
- **规则：** 文章中出现的 URL 必须在 curated items 中存在
- **阈值：** 0 个编造 URL
- **修复：** 删除不存在的 URL

### 2. 空洞表述检测
- **规则：** "值得关注""意义深远""引发热议""广泛关注"等无信息量词语
- **阈值：** <= 3 处
- **修复：** 用具体事实替换

### 3. 口播稿时长校验
- **规则：** 总时长 = hook + overview + deep_items + quick_items + closing
- **阈值：** 180-300 秒
- **修复：** 调整各段时长分配

### 4. editorial 字段长度
- **规则：** observation / evidence / judgment / prediction 每段至少 30 字
- **修复：** 补充内容

### 5. deep_items 含数字
- **规则：** 每条 deep item 的 details 必须包含具体数字
- **修复：** 补充数据

### 6. 事实锚定检查
- **规则：** 文章中出现的数字必须在 curated data 中有来源
- **阈值：** 0 个无来源数字
- **修复：** 删除或补充来源

### 7. 日期一致性
- **规则：** 新闻发布时间必须在目标日期 ±36h 窗口内
- **修复：** 移除超期新闻

### 8. 文章-口播稿一致性
- **规则：** 文章和口播稿的标题交集/并集 >= 50%
- **修复：** 对齐内容

## 人工审核检查点

### 采集后

| 检查项 | 标准 | 不达标处理 |
|--------|------|-----------|
| raw_count | > 0 | 检查 RSS 源可达性 |
| sources_ok | >= 10 | 检查 source-health.json |
| 源多样性 | 覆盖 official/academic/media | 补充源 |

### 评分后

| 检查项 | 标准 | 不达标处理 |
|--------|------|-----------|
| auto 条数 | >= 5 | 降低阈值或补充源 |
| 评分分布 | 不应全部集中在某个分数段 | 检查评分规则 |

### 选题后

| 检查项 | 标准 | 不达标处理 |
|--------|------|-----------|
| selected | 8-15 条 | 调整 LLM prompt |
| deep 级别 | >= 1 条 | 检查是否有足够重磅新闻 |
| 来源多样性 | >= 3 个不同源 | 检查是否有源垄断 |

### 生成后

| 检查项 | 标准 | 不达标处理 |
|--------|------|-----------|
| article.md 结构 | 含 hook/速览/深度/快讯/观点 | 重跑生成 |
| script.md 时长 | 180-300s | 调整 prompt |
| 无编造内容 | 所有数据有来源 | 校验会捕获 |

## 反模式清单

### 内容反模式

| 反模式 | 示例 | 修复 |
|--------|------|------|
| 空洞表述 | "这值得关注" | 用数据替换 |
| 无来源数字 | "市场增长了 300%" | 补充来源或删除 |
| 重复信息 | 速览和深度说同一件事 | 速览用一句话，深度展开 |
| 缺少对比 | "发布了新模型" | 补充"比上一代提升 X%" |
| 缺少影响 | "获得了融资" | 补充"客户包括 X，海外收入占 Y%" |

### 结构反模式

| 反模式 | 示例 | 修复 |
|--------|------|------|
| 深度条目无数字 | details 只有定性描述 | 补充具体数据 |
| editorial 太短 | 每段 < 30 字 | 展开分析 |
| 口播稿超时 | 总时长 > 300s | 压缩 quick_items |
| 标题不一致 | 文章和口播稿标题差异大 | 统一标题 |
