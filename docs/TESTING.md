# AI 日报 — 人工审核测试点

> 本文档列出每次日报生成后需要人工核实的关键环节。标记为 [必审] 的项目直接影响发布质量。

---

## 1. 事实核查 [必审]

### 1.1 深度报道（deep_items）逐条核实

对每条 deep_items，执行以下检查：

- [ ] 点击来源链接，确认页面可访问
- [ ] 确认页面上的核心事实与日报描述一致（公司名、人名、产品名）
- [ ] 确认日报中的具体数字（参数量、融资额、百分比）与原文一致
- [ ] 确认页面发布日期与日报日期匹配

**重点关注**：LLM 在"为什么重要"和"意味着什么"段落中可能编造原文没有的数字或推断。对比原文，标记任何原文中不存在的具体数字。

### 1.2 重要动态（important_items）抽查

- [ ] 随机抽查 2-3 条，点击链接确认内容一致
- [ ] 检查是否有"无来源数据"（日报中的数字在原文中找不到）

### 1.3 快讯（brief_items）抽查

- [ ] 确认链接可点击
- [ ] 确认核心事实无误

---

## 2. 时效性核查 [必审]

### 2.1 日期一致性

- [ ] 打开 `output/<date>/curated.json`，逐条检查 `published_at` 字段
- [ ] 确认所有入选条目的日期为目标日期（允许前一天 18:00 UTC 之后的时区容差）
- [ ] 特别关注 The Verge 来源：RSS pubDate 可能是 updatedAt，需点击原文确认实际发布日期

### 2.2 旧新闻检测

- [ ] 检查是否有"昨天已经报道过的新闻"再次出现
- [ ] 如果发现重复，检查 `output/index.json` 中是否已有记录

---

## 3. 编辑观点审核 [必审]

- [ ] 观点是否基于今日具体新闻？（不是泛泛的趋势总结）
- [ ] 是否引用了具体事实作为证据？
- [ ] 判断是否可反驳？（不是"AI 行业正在快速发展"这种废话）
- [ ] 预测是否合理？（有依据，不是凭空推测）
- [ ] 是否有重复速览标题的段落？（应删除）

---

## 4. 格式和链接检查

- [ ] 文章是否包含所有必需板块：今日速览、重磅深度、重要动态、快讯、编辑观点
- [ ] 所有来源链接是否完整（无空括号 `()` ）
- [ ] 口播稿时长是否在 3-5 分钟范围内
- [ ] 文章与口播稿的新闻选取是否一致

---

## 5. 信源健康检查

- [ ] 查看 `data/source-health.json`，是否有源连续失败 3 天以上
- [ ] 查看 `output/<date>/manifest.json` 的 `sources.failed`，确认失败源

---

## 6. 快速发布检查清单

每次发布前，用 3 分钟完成：

1. 打开 `article.md`，通读一遍
2. 点击 2-3 个来源链接，确认可访问且内容一致
3. 检查编辑观点是否有立场、有证据
4. 确认无旧新闻混入（查看 curated.json 的 published_at）
5. 确认 manifest.json 中 validation_passed: true

---

## 7. 用户反馈收集

在文章底部可添加反馈表单，收集：
- 这期日报最有价值的 1 条新闻
- 这期日报最不值得报道的 1 条新闻
- 编辑观点是否同意？（是/否/部分）
- 整体评分 (1-5)

反馈数据存储在 `output/<date>/feedback.json` 的 `user_feedback` 字段中，可用于：
- 调整评分权重
- 优化 Prompt 示例库
- 改进信源配置
