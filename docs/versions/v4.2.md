# AI 日报 — v4.2 版本文档

> 本文档是项目的「控制面板」。它帮你理解系统在做什么、怎么用、下一步去哪。每次版本更新时创建新文档，旧版本保留在 `docs/versions/` 目录。

---

## 一句话总结

> 从 15+ 信源持续采集 AI 新闻，存入 SQLite 知识库；每天由 Agent 读取知识库，选题、写文章、写口播稿，产出日报。

---

## 系统是什么

AI 日报不是一个「脚本」，而是一个**双 Runtime 内容智能系统**：

```
Ingestion Runtime（持续运行，纯代码，不需要 AI）
  ↓ 采集 → 归一化 → 验证 → 评分 → 去重 → 入库
SQLite Event Repository（持续增长的知识库）
  ↓
Editorial Runtime（每天一次，Agent 驱动，需要 AI）
  ↓ 读取 → 选题 → 写文章 → 写口播稿 → 渲染 → 校验 → 归档
output/<date>/（产出物）
```

**核心价值：** Repository 持续增长。第 365 天的知识库是不可复制的竞争壁垒。

## 当前架构

```
Host → Runtime → ExecutionSession → ExecutionGraph → Task
                                                       ↓
                                               ExecutionContext
                                               ├── Resources（只读）
                                               └── Scope
                                                   ├── events（Repository + ReadModel）
                                                   ├── policyEngine（Policy + Rule）
                                                   └── inference（LLM 调用）
```

### 分层说明

| 层 | 职责 | 当前状态 |
|---|---|---|
| **Host** | Runtime 与外部世界的接口 | ✅ ClaudeHost + NodeHost |
| **Runtime** | 执行 ExecutionGraph | ✅ 纯 Node.js |
| **Task** | 每个步骤的实现 | ✅ 14 个 Task（Ingestion 7 + Editorial 7） |
| **Policy** | 纯业务规则（无 IO） | ✅ 4 个 Policy + 8 个 Rule |
| **Repository** | 写模型（INSERT OR IGNORE） | ✅ SQLite |
| **ReadModel** | 读模型（findByWindow/findByEntity） | ✅ SQLite |
| **Skill** | Agent 入口（薄） | ✅ /daily 触发 Editorial |

### 关键设计决策

1. **Ingestion 不需要 AI** — 全部是确定性代码，直接 execSync 调用脚本
2. **Editorial 是 Skill 驱动** — Agent 自己就是 LLM，不需要 API 调用
3. **Task 负责数据组装，Policy 只做纯计算** — Policy 不碰 IO
4. **Repository（写）+ ReadModel（读）** — 纯 CQRS
5. **content_hash UNIQUE** — SQLite 原子去重，INSERT OR IGNORE

## 如何使用

### 日常操作

```bash
# 采集新闻（手动，或设置 cron 每 5 分钟）
node scripts/run-ingestion.mjs

# 生成日报（在 Claude Code / Cursor / Gemini CLI 中说）
"生成今日日报"
# 或
/daily
```

### 查看数据

```bash
# 查看 SQLite 中有多少事件
sqlite3 data/events.db "SELECT COUNT(*) FROM events"

# 查看今天的事件
sqlite3 data/events.db "SELECT title, rank_total FROM events WHERE effective_at >= '2026-06-25' ORDER BY rank_total DESC LIMIT 10"

# 查看某个实体的历史
sqlite3 data/events.db "SELECT title, effective_at FROM events e JOIN event_entities ee ON e.id = ee.event_id WHERE ee.entity = 'OpenAI' ORDER BY effective_at DESC"

# 查看日报产出
ls output/<date>/
cat output/<date>/article.md
cat output/<date>/script.md
```

### 信源管理

编辑 `scripts/config.mjs` 的 `RSS_SOURCES` 数组。

### 评分调整

编辑 `scripts/config.mjs` 的 `SCORING` / `ENTITY_WEIGHTS` / `EVENT_TYPE_WEIGHTS`。

### Prompt 调整

编辑 `prompts/v1/` 下的模板文件。

## 已知限制

1. Editorial 由 Agent 驱动，质量取决于 Agent
2. RSS 依赖源的可用性和时间格式质量
3. SQLite 单机，不适合多机部署
4. Ingestion 需要手动或 cron 触发
5. 无反馈闭环
