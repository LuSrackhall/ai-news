# v4.4 — Event 聚类 + 周报 + 反馈收集

## 概述

v4.4 在 v4.2 双 Runtime 架构基础上，新增三大能力：

1. **Event 聚类** — Ingestion 阶段自动将相似事件归类
2. **Weekly Pipeline** — 按周聚合事件，生成周报
3. **反馈收集** — 记录用户交互数据（只存不学）

## 聚类算法

ClusterPolicy 使用三重匹配（满足任一即归类）：

1. **实体重叠度 ≥ 0.5** — 交集 / 并集
2. **事件指纹相同** — Entity|EventType|Keywords|Week 哈希
3. **标题 bigram 相似度 ≥ 0.7** — 字符级 bigram Jaccard

Cluster 标题 = 该 Cluster 中 rank_total 最高的 Event 标题。

## Ingestion 流程（更新后）

```
Collect → Normalize → Verify → ExtractEntities → ClusterEvents → Score → Dedup → Store
```

## Weekly Pipeline 流程

```
LoadWeekEvents → AggregateByCluster → GenerateWeeklyArticle → RenderWeekly → ArchiveWeekly
```

输出：`output/weekly/<start>_<end>/` (article.md + script.md + manifest.json)

## 设计原则

- **Runtime 边界不变** — v4.4 不修改 Runtime 核心，只新增 Task + Pipeline
- **聚类算法可替换** — ClusterPolicy 是纯函数，v4.5 可用 LLM 替换
- **反馈只存不学** — feedback 表存原始数据，v4.5 Learning Engine 消费
- **中文实体增强** — 新增 36氪、智谱、百川等中文 AI 生态实体
