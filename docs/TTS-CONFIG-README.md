## 🚀 快速开始

### 1. 首次配置
```bash
# 复制模板
cp .env.example .env

# 编辑配置（至少需要填入 API Key）
vim .env

# 或使用参考卡快速配置
cat .env.cheatsheet
```

### 2. 配置 API Key 池
在 `.env` 中添加多个 Key（逗号分隔）：

```bash
MIMO_API_KEY_POOL=tp-key1,tp-key2,tp-key3
```

**获取 Key**：
- 访问 https://mimo.mi.com
- Token Plan → Console
- 创建 3-5 个 Key

### 3. 选择音色模式

#### 模式1：预设音色（简单）
```bash
PODCAST_MIMO_TTS_MODEL=mimo-v2.5-tts
PODCAST_MIMO_TTS_MALE_VOICE=白桦
PODCAST_MIMO_TTS_FEMALE_VOICE=冰糖
```

#### 模式2：自定义音色（推荐）⭐
```bash
PODCAST_MIMO_TTS_MODEL=mimo-v2.5-tts-voicedesign
PODCAST_MIMO_TTS_FEMALE_VOICE_DESC=一个28岁的知性女性，声音温柔如水
PODCAST_MIMO_TTS_MALE_VOICE_DESC=一个30岁的男性，声音温润磁性
```

### 4. 运行合成
```bash
# 加载环境变量
export $(cat .env | grep -v '^#' | grep -v '^$' | xargs)

# 运行合成
TTS_PROVIDER=mimo bash scripts/tts/synthesize.sh --force output/production/ai/2026-06-29/script.json
```

## 📖 音色设计指南

### 音色描述要素

1. **年龄**
   - 22岁（年轻活力）
   - 28岁（知性成熟）
   - 35岁（优雅稳重）
   - 40岁（沉稳大气）

2. **音色特点**
   - 温柔如水 / 清澈悦耳
   - 甜美清脆 / 银铃般
   - 温润磁性 / 大提琴般
   - 浑厚有力 / 播音员般

3. **语速**
   - 较慢：娓娓道来
   - 适中：节奏平稳
   - 较快：干脆利落
   - 很快：节奏明快

4. **风格**
   - 专业 / 亲切 / 活泼 / 沉稳

## 🔧 Key 池机制

### 工作原理
1. **初始化**：加载所有 Key 到池中
2. **轮换**：每次调用使用下一个 Key（Round-Robin）
3. **重试**：遇到 429 限流时自动切换 Key
4. **恢复**：所有 Key 都限流后等待 60 秒重试

### 优势
- ✅ 不会因单个 Key 限流而失败
- ✅ 自动负载均衡
- ✅ 无缝切换，用户无感知

### 最佳实践
- 配置 3-5 个 Key
- 定期检查 Key 使用情况
- 可混合使用不同集群的 Key（中国/新加坡）

## 🛠️ 故障排除

### 429 Too Many Requests
**原因**：API 限流  
**解决**：
1. 配置多个 Key（推荐）
2. 等待 10-15 分钟
3. 检查 Key 是否有效

### 401 Invalid API Key
**原因**：Key 无效或过期  
**解决**：
1. 检查 Key 是否正确
2. 确认 Key 所属集群
3. 重新生成 Key

### 音色不符合预期
**原因**：描述不够具体  
**解决**：
1. 增加更多描述要素
2. 使用生动的比喻
3. 尝试不同组合
